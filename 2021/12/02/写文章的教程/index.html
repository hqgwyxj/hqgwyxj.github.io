写文章的教程

2021-12-02- 2021-12-06
592- 2m
教程
FDYF

  1. 每次写文章前，先在博客的仓库（就是文件夹）里打开git bash，输入git pull（这一步是为了让你把最新的网站内容拉取也就是下载下来，然后再在此基础上写新的文章）

  2. 我觉得你目前不需要写除了文章之外的其他东西（比如页面），所以每次就hexo new 文件名 就可以了，注意文件名后面不要写.md

  3. new之后，在h_files文件夹里找到source，再再source里找到_posts，你会发现那里面已经有一个题目如上的md文件了，打开它

  4. 你会发现这个文件里已经有一些内容了。你需要在两条虚线之间另起一行写上

    author: barricata

注意冒号和你的名字之间有空格！这一行是为了显示这篇文章的作者是你
5. 再另起一行，写上

password: 你的密码

密码你可以自己定（也可以不写），但建议所有文章设相同的密码，要不然我感觉你容易忘。不用不告诉我密码，反正我在文件里都能看到:)
6. 然后你就可以在第二道虚线下面开始写正文了！你可以用markdown对文章进行排版，markdown教程：https://github.com/younghz/Markdown
7. 写完之后保存文章，然后用git bash（位置还是和刚刚一样，在博客的主文件夹）输入：
8. hexo clean（意思是清除各种没用的文件）
9. hexo g（好像是生成网页）
10. hexo s（在本地浏览器打开，这时你就可以在浏览器输入localhost:4000来查看网页）
11. 如果网页没有问题的话，你需要再次git pull（万一我在你写文章的时候更新了新文章，那你上传上去之后可能就会出现冲突（呃其实我也不知道会不会，不过保险起见吧）
12. hexo d（把网页部署到github pages，这是指让网页在GitHub上显示，和备份不是一回事）
13.同样的git bash：

git add .（英文句号）
git commit -m "注释"（注释就是写写你这次改动了什么，几个字就行）
git push -u origin master（把代码上传到h_files上）
  1. 看一下github上h_files仓库里的文件是不是最新的，是的话就完成了
